Технологія доступу до даних ADO.NET
Вступ.
Технологія ADO поєднує засоби програмування платформи .NET (C#) та
системи управління базами даних (СУБД), тобто надає доступ до даних
з БД у програмному коді.
Технологія містить універсальні засоби, які годяться для різних СУБД
(PostgresSQL, MSSQL/SqlServer, MySQL, Oracle ХЕ, тощо)

C#           ADO         Provider            DB
SELECT 1  -------------------------------->   1
          <-----------------------------------

Provider - драйвер, коннектор - спец. програми для переведення загальних
команд ADO у конкретні команди DBMS

Створення проєкту
1. Новий проєкт WPF
2. Створюємо БД : ПКМ по проєкту - Add - new item - [Data] Service-based DB
    "ADO-P12.mdf" - створити
3. DblClick по "ADO-P12.mdf" - відкривається Server Explorer і показує
    створену БД
4. У Server Explorer фокусуємо БД (або ПКМ - Properties)
     у вікні властивостей знаходимо "Connection String" та копіюємо значення:
Data Source=(LocalDB)\MSSQLLocalDB;AttachDbFilename=C:\Users\_dns_\source\repos\ADO-P12\ADO-P12.mdf;Integrated Security=True
5. Вставляємо це значення як константу Програми (App.xaml.cs)

6. У відповідності до обраної СУБД (у нас MSSQL) додаємо постачальника
    Як правило, це робиться через NuGet
    Для MSSQL provider - Microsoft.Data.SqlClient

Робота з БД
1. Підключення (до БД) - основний об'єкт, що забезпечує передачу команд до БД
    та результатів її виконання до програми
    !! Особливість .NET у тому, що створення об'єкту-підключення
    connection = new(App.ConnectionString);
    не підключає БД, тільки створює об'єкт
    Для відкриття підключення потрібна додаткова команда
    connection.Open();
    !! Підключення до БД належать до некерованих ресурсів, відповідно, їх
    слід руйнувати власними засобами
    а) вживати блок using
    б) викликати метод Dispose()

2. Виконання запитів - передача тексту SQL до СУБД
